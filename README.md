# Barevid

**Barevid** 是一套「**几乎纯文本 → 自动放映 + 配音 → 可导出小视频**」的开源自托管方案。  
仓库内 Web 应用目录仍名为 **`SlideForge/`**（与现有部署习惯兼容）；生产环境目录、镜像名继续叫 SlideForge 也完全合理——**产品对外可称 Barevid，运维命名保持 SlideForge 不变**。

---

## 这和「一键 AI 生成视频」有什么不一样？

| 路线 | 典型产品在做什么 | 成本与可控性（量级） |
|------|------------------|----------------------|
| **像素级文生视频** | 直接生成连续画面（镜头、运动、光影） | 商业 API 常见 **按秒计费**，单价高；改一句台词往往要整段重跑。 |
| **Barevid 这条路线** | 用 **结构化讲稿 + 分页幻灯放映 + TTS** 合成「讲解型 / 播客型」短片 | 主要消耗 **大模型整理文案** + **语音合成**；**不按视频秒数向「生成画面」厂商付钱**。 |

### 市面价格大概在什么量级？（以官网为准，会变动）

以下为 **2025–2026 年前后** 公开信息里的**数量级**感受，便于对比；下单前请务必以各平台**当前官网**为准。

- **Runway（API）**：[官方定价说明](https://docs.dev.runwayml.com/guides/pricing/) 中，不同模型按 **credits/秒** 计，折算下来常见是 **每秒若干美分量级** → **每分钟美元级** 很常见。
- **Pika、即梦、可灵等**：多为 **订阅 + 积分/条数**，或 **按段/按时长** 计费；面向「生成镜头画面」的场景，**同样一条成片反复试错时，总成本会上去**。

**Barevid 的成本结构不同**：你没有为「每一帧假视频」付 GPU 视频生成费，而是为 **文本 token** 和 **TTS 字符/次数** 付费（并可替换为本地 Coqui 等）。在典型配置下，**约 5 分钟成片的 API 成本可落在 ¥0.05～¥0.1 量级**（随台词长短、是否多次重生成、所选模型单价而浮动；若全程本地 TTS，还可更低）。

## 你为什么可能更在意 Barevid？

1. **成本曲线友好**  
   成片时长主要来自 **朗读与翻页节奏**，而不是「每多一秒画面就多烧一档视频模型钱」。适合长讲稿、课程、产品说明、内部培训等 **信息密度高、画面不必电影级** 的场景。

2. **「逐页 surgical 编辑」**  
   文稿被拆成 **分段幻灯 / 分段音频**（如 `001.mp3` …）。改某一页的旁白或节奏，**不必从头生成整条视频**；前端可逐段调整，再单独重导出受影响部分——**操控自由度**远高于「一条 prompt 出一镜到底」的黑盒流程。

3. **管线透明、可替换**  
   当前实现：**杂乱文本 → DeepSeek 整理成两层结构（大标题 / 小标题）→ 豆包 TTS 按段合成 → React 放映页按时间轴播放 →（可选）Playwright + ffmpeg 录屏导出**。LLM 与 TTS 均可换供应商或本地化，**开源后社区可以接 Coqui、其他端侧模型等**。

4. **远程导出 Worker**  
   `worker/` 可独立部署，轮询后端导出任务，在隔离进程里跑 Playwright + ffmpeg，**不把重负载绑死在 API 进程上**。

## 实现上「巧」在哪里？

- **用「幻灯片 + 时间轴」代替「生成视频流」**：画面是可控的 Web 放映，质量稳定、可预览、可逐页改；导出时再编码成 mp4。  
- **字级/段级时长回写**：TTS 结果落库，前端按真实音频时长推进，避免口型与字幕和声音对不齐的玄学问题。  
- **导出与业务解耦**：导出任务入队，Worker 用约定密钥拉活——适合单机开发到 small cluster 的平滑扩展。

---

## 这项目有价值吗？适合开源吗？

**有价值，但是「垂直场景」的价值，不是万能替代品。**

- **更适合**：教程、播客式解说、产品更新说明、文档朗读视频、开发者自制内容、可自托管的「文本驱动讲解视频」工具链。  
- **不试图替代**：Runway / Pika 那类 **电影感镜头、复杂运动、纯画面创意** 的生成——路线不同，比较维度也应不同。

开源意义在于：把 **低成本 + 高可控 + 可替换后端** 整条链路摊开，别人可以 **自托管、改 UI、换 TTS、接企业内网**。对独立开发者和小团队，这比「只能调用黑盒 SaaS」更有长期价值。

## 仓库结构

```
barevid/
├── SlideForge/          # Web：FastAPI 后端 + Vite/React 前端（详细说明见内层 README）
├── worker/              # 视频导出 worker（轮询任务 + Playwright + ffmpeg）
└── README.md            # 本文件
```

更细的依赖、环境变量与启动命令见 **[SlideForge/README.md](./SlideForge/README.md)**。

## 快速开始（指向内层文档）

1. 配置并启动 **SlideForge/backend**（Python）与 **SlideForge/frontend**（Node）。  
2. 按需配置 **worker/.env** 与后端的导出相关环境变量，启动 `worker_export_video.py`。  

具体步骤以 `SlideForge/README.md` 为准。
